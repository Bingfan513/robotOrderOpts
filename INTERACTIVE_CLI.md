# 交互式 CLI 功能指南

本文档说明了即将在下一阶段实现的交互式命令行界面功能。

## 功能概述

交互式 CLI 模式允许用户通过命令与机器人订单系统进行实时交互。

## 命令参考

### 订单命令

```bash
order [regular|vip]  # 创建订单

示例:
  order regular      # 创建普通订单
  order vip         # 创建VIP订单
```

### 机器人管理命令

```bash
robot [add|remove|list]  # 管理机器人

示例:
  robot add         # 添加新机器人
  robot remove 1    # 移除机器人1
  robot list        # 列出所有机器人
```

### 系统命令

```bash
status              # 显示当前系统状态
stats               # 显示详细统计信息
clear               # 清空屏幕
help                # 显示帮助信息
exit / quit         # 退出程序
```

## 使用示例

### 启动交互式模式

```bash
# 使用默认配置（2个机器人）
./robot-order-system -i

# 使用自定义机器人数量
./robot-order-system -i -robots 5
```

## 时间戳说明

所有输出都包含 `[HH:MM:SS]` 格式的时间戳：

- `HH` - 小时（00-23）
- `MM` - 分钟（00-59）
- `SS` - 秒（00-59）

示例：`[15:04:05]` 表示下午 3:04:05

## 命令执行流程

```
用户输入 → 命令解析 → 验证 → 执行 → 更新系统 → 显示结果
                                    ↓
                            写入 result.txt
                            （带时间戳）
```

## 注意事项

1. **时间戳准确性**
   - 时间戳来自系统时钟
   - 精度为秒级
   - 用于追踪事件顺序

2. **输出双写**
   - 所有输出同时发送到控制台和 result.txt
   - 确保可追溯性
   - 便于审计和调试

3. **并发安全**
   - 所有对共享资源的访问都是线程安全的
   - 机器人在独立的 goroutine 中运行
   - 订单队列使用 Mutex 保护
